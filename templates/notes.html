Documentação:

Documentação de alguns métodos, sua logica e localização para ser de mais fácil leitura

  1. Estrutura do projecto:
    • root:
      - ficheiro index.html - corre algumas validações do browser. - ficheiro index.js - bootstrap da aplicação + funcões não categorizadas ( que n valia a pena agrupar ).
    • templates: Ficheiros de template html que são carregados dinamicamente e postos em cache (ficheiro de load e cache: /scripts/modules/cache/template/index.js)

    • scripts: Pasta dos ficheiros javascript

      - constants.js - constantes - variaveis globais ( foi o nome que dei inicialmente pois tencionava encapsular a aplicacao mas não vale a pena)

      --\warnings: vinha com o ficheiro base do bootstrap (http://getbootstrap.com) - não é da minha autoria

      --\workarounds: vinha com o ficheiro base do bootstrap (http://getbootstrap.com) - não é da minha autoria

      --\modules: tentei dividir a aplicação em namespaces para conseguir organizar melhor o meu código e ter ficheiros mais legiveis e mais pequenos. Estes módulos têm comentado na primeira linha as dependencias

      ----\cache: Sistema simples para fazer cache da aplicação para não estar a fazer vários pedidos desnecessarios

      ----\game: Sistema do jogo

      ----\aux: Funções auxiliares ( normalmente as usaria um framework para fazer o trabalho destas)

      ----\router: Sistema que carrega as tabs e inicia os métodos de bootstrap do Objecto responsável por elas ( neste caso tab game tem o objecto Game )

      ----\ui: Funções de user interface (modal e scroll)
